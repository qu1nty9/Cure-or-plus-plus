# OpenAI GPT-5.5 Full Provider Run v0.1

## Run

- Provider: OpenAI
- Model: `gpt-5.5`
- Endpoint: Responses API
- Storage flag: `store=false`
- Prompt pack: `reports/vlm_api_track_v01_prompt_pack.jsonl`
- Rows: 210 total, 30 clean and 180 real-transfer
- Temperature: omitted because this model rejects the parameter
- Reasoning effort: `low`
- Output cap: initial full pass used `max_tokens=64`
- Sanitized merged response file: `reports/vlm_api_track_v01_responses_openai_gpt_5_5_merged_full.jsonl` (local, ignored by Git)

## Retry/Merge Policy

The first `max_tokens=64` full pass completed all 210 rows but left 8 empty
responses because the output budget was consumed by hidden reasoning tokens.
Those rows were retried with `max_tokens=128`; one remaining empty row was
retried with `max_tokens=256`. The final merged response file keeps the
original full-pass rows except where a later retry provided a non-empty answer.

The merge was generated by:

```bash
python3 scripts/merge_vlm_response_retries.py \
  --base reports/vlm_api_track_v01_responses_openai_gpt_5_5_low64_full.jsonl \
  --retry reports/vlm_api_track_v01_responses_openai_gpt_5_5_low128_retry_unparseable.jsonl \
  --retry reports/vlm_api_track_v01_responses_openai_gpt_5_5_low256_retry_last.jsonl \
  --output reports/vlm_api_track_v01_responses_openai_gpt_5_5_merged_full.jsonl
```

label is `calcium_bottle`, with clean accuracy `0.6667` and real-transfer

## Interpretation

GPT-5.5 validates the flagship hosted-provider path, but it required a
larger output budget than GPT-5.4-mini because hidden reasoning could consume
the full visible-output cap. After targeted retries, the row is fully parseable
and reaches strong real-transfer accuracy, but it does not exceed GPT-5.4-mini
on this 210-row object-recognition prompt pack.
